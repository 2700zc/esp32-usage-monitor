# 项目名称
ESP32 语音输入终端（AI Voice Keyboard）

---

# 一、项目目标（What）

我要做一个设备：

👉 按下 ESP32 右侧按键  
👉 开始录音  
👉 说话内容实时/结束后转成文字  
👉 自动输入到电脑当前输入框  
👉 再按一次停止录音  

本质是：

> 一个“AI语音键盘”

---

# 二、系统架构（How）

## 架构必须如下：

ESP32 负责：

- 按键检测（开始/停止录音）
- I2S 麦克风采集音频
- WiFi连接
- WebSocket发送PCM音频数据
- OLED/AMOLED状态显示（可选）

---

PC 负责：

- WebSocket Server 接收音频流
- 音频缓存与拼接
- 语音转文字（ASR）
  - 使用 faster-whisper（优先）
- 将文本自动输入到当前系统光标位置

---

# 三、技术约束（Constraints）

## ESP32侧：

- 必须使用 ESP-IDF（优先）
- 音频格式：
  - 16kHz
  - 16bit PCM
  - mono
- 使用 WebSocket（禁止 HTTP轮询）
- 不能做语音识别

---

## PC侧：

- Python实现
- 必须支持实时接收音频
- 使用 faster-whisper（本地模型，不用云API）
- 使用 keyboard 或 pyautogui 实现自动输入

---

# 四、核心交互流程（Flow）

## 1. 按键触发

ESP32：

- 按下按钮 → 开始录音
- 再按一次 → 停止录音

---

## 2. 录音过程

ESP32：

- 持续采集 I2S 音频
- 实时通过 WebSocket 发送 PCM 数据

---

## 3. PC处理流程

PC：

- 接收音频流
- 缓存成 wav
- 触发 Whisper 识别
- 输出文本

---

## 4. 输入到系统

PC：

- 自动检测当前焦点输入框
- 将文本“模拟键盘输入”

---

# 五、UI状态（ESP32屏幕）

必须实现状态反馈：

## 待机状态
- READY
- WiFi connected

## 录音状态
- 🎤 Listening...

## 识别状态
- 🧠 Processing...

## 完成状态
- ✅ Done

---

# 六、模块拆分（必须按这个写代码）

## ESP32模块：

- button.c
- audio_i2s.c
- websocket_client.c
- state_ui.c

---

## PC模块：

- server.py（WebSocket服务）
- audio_buffer.py（音频缓存）
- asr_whisper.py（语音识别）
- keyboard_input.py（自动输入）

---

# 七、性能要求

- 延迟 < 2秒（停止说话到出字）
- 支持连续长时间录音
- 支持中文 + 英文混合
- 支持断句识别

---

# 八、禁止事项（Important）

❌ 不允许 ESP32 做语音识别  
❌ 不允许使用云API（除非我后续说）  
❌ 不允许HTTP轮询传输音频  
❌ 不允许一次性上传整段音频（必须支持流式）

---

# 九、优先级（Important Order）

请按以下顺序实现：

1. ESP32 → PC WebSocket 通信（先验证通）
2. ESP32 I2S 录音
3. PC 接收音频并保存 wav
4. Whisper离线识别
5. 自动输入文本
6. UI状态优化
7. 流式识别优化（最后做）

---

# 十、最终效果（Goal）

完成后设备行为：

- 按键一次 → 开始说话
- 再按一次 → 自动打字到电脑
- 整体体验像“语音输入法 + 物理键盘”
